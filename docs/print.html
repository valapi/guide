About

valorant.ts is a NodeJS package that make more easier to use Valorant API

  • All-in-one package
  • Support Multi-factor Authentication (2FA)
  • 5 Packages:
    • lib
    • api-wrapper
    • riot-api
    • riot-client
    • valorant-api.com

Install

PNPM:

pnpm add valorant.ts

Yarn:

yarn add valorant.ts

NPM:

npm install valorant.ts

Requirements

  • Node.js (we recommend lastest version for best performance)

Introduced

import { Client } from '@valapi/api-wrapper';

Config

interface Config {
    // UserAgent use when authentication
    userAgent?: string;

    // Region of the Account
    region?: string;

    // Client Settings
    client?: {
        version?: string,
        platform?: {
            "platformType": string,
            "platformOS": string,
            "platformOSVersion": string,
            "platformChipset": string,
        },
    };

    // No throw error if the authentication is failed.
    forceAuth?: boolean;

    // Config for RequestClient (Axios)
    axiosConfig?: AxiosRequestConfig;

    // The time in milliseconds that data will be refreshed.
    expiresIn?: {
        cookie: number,
        token?: number,
    };

    // When Cookie is expired, it will be refreshed.
    selfAuthentication?: {
        username: string | Function,
        password: string | Function,
        verifyCode?: string | number | Function,
    };
}

Client

const ApiClient = new Client(config);

Save

Client --> Save

ApiClient.toJSON();

ApiClient.toJSONAuth(); // recommended for authentication

Save --> Client

Client.fromJSON(config, ApiClient.toJSON());

Client.fromJSONAuth(config, ApiClient.toJSONAuth());

Basic Authentication

Login

First We Need To Login To Valorant Account

await ApiClient.login('Username', 'Password');

But We Have 2 Type Of Account

Multi-Factor

When The Account Have Multifactor You Can't Use Normal Method

After Login The Verify Code Will Send To Mail

await ApiClient.verify(Number);

Advanced Authentication

Beta

Cookie

await ApiClient.fromCookie(cookie?);

Reconnection

its will reconnection if the token or cookie is expired

await ApiClient.reconnect(isForce?);
